# 🃏 OWASP Cornucopia — Contributor Dashboard

A personal dashboard to track and showcase your contributions to [OWASP/cornucopia](https://github.com/OWASP/cornucopia).  
Inspired by [OWASP-BLT/MY-GSOC-TOOL](https://github.com/OWASP-BLT/MY-GSOC-TOOL).

**Live demo:** `https://YOUR-USERNAME.github.io/cornucopia-dashboard/`

---

## 🚀 Setup in 5 steps

### Step 1 — Fork / create repo
Create a new GitHub repository named `cornucopia-dashboard` and push these files.

### Step 2 — Edit `index.html` (your personal info)

Open `index.html` and find the `CONFIG` block near the bottom inside `<script>`.  
Fill in **your** details:

```js
const CONFIG = {
  contributor: {
    name:           "Your Full Name",
    githubUsername: "your-github-username",   // ← IMPORTANT
    bio:            "OWASP Cornucopia Contributor",
    github:         "https://github.com/your-username",
    blog:           "https://yourblog.dev",
    linkedin:       "https://linkedin.com/in/your-profile",
  },
  project: {
    description: "Contributing to the OWASP Cornucopia threat modelling card game.",
    timeline:    "2026 – Present",
    repoUrl:     "https://github.com/OWASP/cornucopia",
  },
  // stats are auto-updated by GitHub Actions — leave as 0
  stats: { commits: 0, pullRequests: 0, mergedPRs: 0, issues: 0 },

  // add your own entries here:
  activity:      [ /* your PRs / commits / issues */ ],
  weeklyUpdates: [ /* week 1, week 2 … */ ],
  blogPosts:     [ /* your articles */ ],
};
```

### Step 3 — Add repository variable

Go to **Settings → Secrets and variables → Actions → Variables → New repository variable**:

| Name | Value |
|------|-------|
| `CONTRIBUTOR_USERNAME` | `your-github-username` |

This tells the workflow which GitHub user to fetch contributions for.

### Step 4 — Enable GitHub Pages

Go to **Settings → Pages → Source → GitHub Actions** → Save.

### Step 5 — Run the workflow

Go to **Actions → Deploy Cornucopia Dashboard → Run workflow**.

Your dashboard will be live at:
```
https://YOUR-USERNAME.github.io/cornucopia-dashboard/
```

---

## 🔄 Auto-updates

The workflow runs:
- **Daily at midnight UTC** — auto-fetches your latest contributions from OWASP/cornucopia
- **On every push** to `main`
- **Manually** via Actions tab

## 📂 File structure

```
cornucopia-dashboard/
├── .github/
│   └── workflows/
│       └── deploy.yml              # Auto-update + deploy to GitHub Pages
├── data/
│   └── github-contributions.json  # Auto-generated by the workflow
├── scripts/
│   ├── fetch-contributions.js      # Fetches your GitHub activity
│   └── inject-stats.js             # Patches stats into index.html
├── index.html                      # ← EDIT THIS with your info
└── README.md
```
